Midi-Kommando:
=============
- Konstante Länge von drei Byte

1 Erstes Byte[7..4] enthalten Status (http://www.midi.org/specifications-old/item/table-1-summary-of-midi-message)
1 Erstes Byte[3..0] enthalten MIDI Channel Number
	- MIDI Channel kann man gleichetzten mit einem Instrument wie z.B. Schlagzeug
1 Erstes Byte, Erkennung am Status, da MSB = 1, bei den folgenden Bytes sollte das nicht so sein

2 Zweites Byte enthält Daten und beginner immer mit einer 0 als MSB (Wertebereich von 0-127)

3 Zweites Byte enthält Daten und beginner immer mit einer 0 als MSB (Wertebereich von 0-127)


Mögliche Empfangsfälle:
======================
1. Alle drei Bytes in richtiger Reihenfolge und korrekt empfangen 	-> Licht ändern
2. Nur ein Byte oder zwei Bytes empfangen (welche davon ist egal)	-> Erkennung über Timeout
3. Ungültiges oder nicht releventes Byte empfangen					-> Erkennung beim Parsen


Timout Funktion:
===============
Wenn die Anzahl der empfangenen Byte Eins oder Zwei beträgt, soll nebenbei ein Zähler
laufen. Erreicht dieser Zähler einen bestimmten Wert, sind die bereits emfangenen Daten
zu verwerfen und es wird auch ein komplett neues MIDI-Kommando gewartet. Trifft ein 
weiteres Byte vor Ablauf des Zählers ein, wird der Zähler gestoppt oder neu gestartet.


Parsen:
======
Wird ein Byte empfangen, wird dieses direkt geparst und ein entsprechender Parse-Zustand
gesetzt. Wurde noch kein Byte empfangen, ist der Parse-Zustand "WAITING".

Erstes Byte:
	Aufteilung nach Status und MIDI Channel Number, jedes Segment separat vergleichen.
	Für die Vergleiche müssen über Defines Vergleichwerte vorgegeben werden, die ggf. 
	bei weiteren Tests schnell geändert werden können.
	Stimmen die beiden Segmente mit den Vergleichswerten überein, wird der Parse-Zustand
	auf "STATUS_AND_CHANNEL_RECEIVED" gesetzt. Anderfalls ändert sich der Zustand nicht.
	
Zweites Byte:
	Mit diesem Byte wird die Farbe definiert. Für Rot, Grün und Blau muss es somit
	ebenfalls jeweils eine Vergleichsnote (ebenfalls über Defines) geben. Stimmt einer
	der Vergleiche überein, wird sich die Farbe gemerkt und der Parse-Zustand auf
	"NOTE_RECEIVED" gesetzt. Stimmt kein Vergleich überein, wird der Parse-Zustand auf
	"WAITING" zurückgesetzt.
	
Drittes Byte:
	Gibt die Leuchtstärke an. Zu überprüfen gilt, ob das Byte eine '0' im MSB enthält.
	Wenn ja, ist das Byte gültig und die Leuchtstärke kann für die Frabe entsprechend
	übernommen werden. Ggf. kann der Leuchtwert um eine Stelle nach Links geschoben
	werden.	Anschließend wird eine Aktualisierung des Lichtschlauch vorgemerkt. Der
	Parse-Zustand wird zurückgesetzt auf "WAITING". Sind weitere Daten im Empfangsbuffer
	vorhanden, werden diese zunächst geparst.
	

Zahlen und Zeiten:
=================
Dauer eines Bits bei Baudrate von 31250: 32µs
Dauer über Übertragung eines Bytes: 	320µs (10 Bit notwending)
Dauer Empfang von drei Bytes:			960µs

Senden eines Bits per SPI mit 4 MHz:	0.25µs
Senden einer RGB-LED (24 Bits):			  6µs
Senden eines Schlauchs mit 50 LEDs:		300µs


Konfigurierbare Parameter:
=========================
MIDI Channel:			0-15	(1 Byte)
MIDI Note Red:			0-127	(1 Byte)
MIDI Note Green:		0-127	(1 Byte)
MIDI Note Blue:			0-127	(1 Byte)
MIDI Note Off Skip:		0-1		(1 Byte)

No Data Light Red:		0-255	(1 Byte)
No Data Light Green:	0-255	(1 Byte)
No Data Light Blue:		0-255	(1 Byte)
No Data Light Timeout:	1-255	(1 Byte) // in 200 ms Schritten

Permanent Light Red:	0-255	(1 Byte)
Permanent Light Green:	0-255	(1 Byte)
Permanent Light Blue:	0-255	(1 Byte)
